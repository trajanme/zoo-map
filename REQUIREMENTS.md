# zoo-map 要件定義書

> **改訂履歴**
> - v1 (2026-07): MVP(架空2園、矩形エリア)
> - v2 (2026-07): フェーズ2 — 実在動物園(王子動物園)の収録とポリゴンエリア対応(§7)

## 1. 背景・目的

動物園の紙の園内マップは見にくく、目当ての動物がどこにいるのか探しづらい。
「動物園ごとの園内マップ」と「動物の検索・詳細情報」をスマートフォンで見やすく提供する Web アプリを作る。

旧実装(Vue 2.5 CDN 版 `index.html`)は書籍のサンプル写経の段階で頓挫していたため、
本来の目的に沿って要件を再定義し、作り直す。

## 2. MVP スコープ

公開(デプロイ)可能な状態を MVP のゴールとする。

### 2.1 画面と機能

| # | 画面 | パス | 機能 |
|---|------|------|------|
| 1 | 動物園一覧(トップ) | `/` | 収録している動物園をカード形式で一覧表示。タップでその動物園のマップへ |
| 2 | 園内マップ | `/zoos/:zooId` | SVG ベースの園内マップを表示。エリア(ゾーン)を色分けし、動物の位置にピンを表示。ピンをタップするとポップアップで動物名を表示し、詳細ページへ遷移できる |
| 3 | 動物一覧・検索 | `/zoos/:zooId/animals` | その動物園の動物を一覧表示。名前(漢字・かな)のインクリメンタル検索と、エリアによる絞り込みができる |
| 4 | 動物詳細 | `/zoos/:zooId/animals/:animalId` | 動物の名前・よみがな・分類・説明・いるエリアを表示。「マップで場所を見る」からマップ上の該当ピンにフォーカスした状態で遷移できる |

### 2.2 非機能要件

- **モバイルファースト**: スマホ(幅 375px)で崩れず操作しやすいこと。PC 幅でも破綻しないこと
- **静的ホスティングで公開可能**: バックエンド不要。GitHub Pages にデプロイできること(GitHub Actions ワークフローを同梱)
- **オフラインデータ**: 動物園・動物データは静的 JSON(またはそれに相当するモジュール)として同梱

### 2.3 スコープ外(今回はやらない)

- ログイン・認証、動物の新規登録などの管理機能(旧コードのログイン/登録デモは削除する)
- バックエンド API・データベース
- 実在動物園の公式マップ画像の利用(権利上の問題を避け、サンプルデータは架空の動物園とする)
- 現在地測位・園内ナビゲーション
- 動物の写真(プレースホルダー画像・絵文字で代替)

## 3. 技術構成

| 項目 | 採用技術 | 備考 |
|------|----------|------|
| フレームワーク | Vue 3 (Composition API) | 旧 Vue 2.5(EOL)から移行 |
| ビルド | Vite | |
| ルーティング | Vue Router 4 | ハッシュモード(GitHub Pages でリロード 404 を避けるため) |
| マップ描画 | インライン SVG | 外部マップライブラリは使わない。viewBox `0 0 1000 700` 基準 |
| データ | `src/data/` 配下の静的データ | スキーマは §4 |
| デプロイ | GitHub Actions → GitHub Pages | `vite.config` の `base` をリポジトリ名に設定 |

旧 `index.html` は削除する(git 履歴に残るため退避不要)。

## 4. データモデル

```ts
interface Zoo {
  id: string            // 例: "minato-zoo"
  name: string          // 例: "みなと動物園"
  description: string
  areas: Area[]
  animals: Animal[]
}

interface Area {
  id: string            // 例: "savanna"
  name: string          // 例: "サバンナゾーン"
  color: string         // マップ塗り分け用 (例: "#c8e6c9")
  rect: { x: number; y: number; w: number; h: number }  // viewBox 1000x700 内の矩形
}

interface Animal {
  id: string            // 例: "african-elephant"
  name: string          // 例: "アフリカゾウ"
  kana: string          // 例: "あふりかぞう" (検索用)
  species: string       // 分類 (例: "哺乳類 / 長鼻目")
  emoji: string         // 写真代わりの絵文字 (例: "🐘")
  description: string
  areaId: string        // 所属エリア
  position: { x: number; y: number }  // マップ上のピン位置
}
```

## 5. サンプルデータ

架空の動物園 2 園を収録する。

- **みなと動物園**: 5 エリア程度、動物 10 種程度(ゾウ・キリン・ライオン・ペンギンなど定番動物)
- **もりのおか動物園**: 4 エリア程度、動物 8 種程度(レッサーパンダ・ニホンザルなど。旧データの動物を引き継ぐ)

説明文は各動物 2〜3 文程度の紹介文とする。

## 6. 受け入れ基準(MVP)

1. `npm install && npm run build` が成功する
2. `npm run dev` で起動し、全 4 画面がルーティング含めて動作する
3. マップのピンをタップ → ポップアップ → 詳細ページ遷移が動作する
4. 動物一覧でかな・漢字どちらの部分一致検索も動作し、エリア絞り込みと併用できる
5. 詳細ページの「マップで場所を見る」で該当ピンが強調表示される
6. 幅 375px でレイアウトが崩れない
7. GitHub Pages 用のデプロイワークフローが同梱され、`base` 設定が正しい
8. README が新構成(セットアップ手順・デプロイ手順)に更新されている

## 7. フェーズ2: 実在動物園の収録とポリゴンエリア対応

### 7.1 目的

矩形エリアの架空データだけでは「実際の動物園で動物を探す」という本来の目的を果たせない。
実在の動物園として **神戸市立王子動物園** を収録し、実際の園内配置により近いエリア表現を可能にする。

### 7.2 データモデルの拡張

```ts
interface Area {
  id: string
  name: string
  color: string
  // 形状は rect / points のどちらか一方を指定する(points があれば優先)
  rect?: { x: number; y: number; w: number; h: number }
  points?: Array<{ x: number; y: number }>   // ポリゴン(viewBox 座標系)
  labelPos?: { x: number; y: number }        // ラベル位置(省略時は図形の中心)
  facility?: boolean                          // true = 動物のいない施設エリア(遊園地・資料館など)
}

interface Zoo {
  // 既存フィールドに加えて
  disclaimer?: string   // 非公式マップである旨などの注意書き。マップ画面に表示する
}
```

- 既存の架空2園(rect のみ)はそのまま動作すること(後方互換)
- `facility: true` のエリアは、マップ上ではグレー系の控えめな配色+ラベルで描画し、
  動物一覧のエリア絞り込み候補には表示しない
- `labelPos` 省略時、ポリゴンは頂点座標の平均をラベル位置とする

### 7.3 実在動物園の収録方針

- 公式園内マップの画像・図面は複製しない。**公開情報に基づく概略の非公式マップ**として
  独自に作図する(位置関係・隣接関係が概ね合っていればよい)
- `disclaimer` に「非公式の概略マップであり、実際の配置・飼育状況とは異なる場合がある。
  最新情報は公式サイトを参照」の旨を記載し、マップ画面上に表示する
- 飼育動物は異動があるため、収録時点の公開情報に基づく代表的な動物に絞る
  (例: ジャイアントパンダのタンタンは 2024 年に死亡しており収録しない)

### 7.4 王子動物園のデータ要件

出典: 公式サイト(kobe-ojizoo.jp)の公開情報・報道等。園はおおむね南北にやや長い区画で、
南端に正面ゲート。以下のエリア構成をポリゴン(一部矩形可)で表現する。

| エリア | 位置(概略) | 代表動物・内容 |
|--------|--------------|----------------|
| 正面ゲート・フラミンゴ広場 | 南端 | フラミンゴ |
| 遊園地 (facility) | 東側 | 動物なし |
| 旧ハンター住宅 (facility) | 北東の高台 | 動物なし(異人館) |
| 動物科学資料館 (facility) | 南西 | 動物なし |
| 円形猛獣舎 | 中央 | ライオン、アムールトラ、ジャガー、シベリアオオヤマネコ |
| 猿舎・類人猿エリア | 中央東寄り | チンパンジー、ニホンザル |
| 放養式動物舎 | 中央西寄り | ホッキョクグマ、カリフォルニアアシカ、ツキノワグマ |
| 草食動物エリア | 北側 | アジアゾウ、アミメキリン、シマウマ、カバ、レッサーパンダ(カバ舎西隣) |
| 動物とこどもの国 | 北西 | コアラ、フンボルトペンギン、カピバラ |

- 動物は上記から 15〜17 種程度を収録。説明文は各 2〜3 文、種別(分類)とよみがなを付ける
- 動物園一覧(トップ)では、架空2園と区別できるよう「実在の動物園(非公式マップ)」で
  あることが分かる表示を添える

### 7.5 受け入れ基準(フェーズ2)

1. ポリゴンエリアがマップ上で正しく描画され、ラベル・色分け・ピン・フォーカスが従来どおり動作する
2. 既存の架空2園の表示・動作に回帰がない
3. 王子動物園がトップに表示され、マップに disclaimer が表示される
4. facility エリアがグレー系で描画され、動物一覧のエリア絞り込みに現れない
5. `npm run build` 成功、375px でレイアウトが崩れない

## 8. フェーズ3: マップのパン・ズーム、個別獣舎、種・個体情報

### 8.1 マップのスクロール(パン)・ズーム対応

- **パン**: マウスドラッグ / タッチドラッグでマップを移動できる
- **ズーム**: マウスホイール / ピンチ操作、および画面上の「＋ / − / リセット」ボタンで拡大縮小できる
- 実装は外部ライブラリを使わず、SVG の viewBox(または transform)操作で行う
- ズーム倍率はおおむね 1x〜8x。パン・ズームはマップ範囲外へ大きく出ないようクランプする
- ピン・獣舎の**タップとドラッグを判別**する(一定距離以上動いたらタップ扱いにしない)
- `?focus=<animalId>` で遷移した場合は、該当ピン付近へ**自動ズーム**(3x 程度)して表示する

### 8.2 動物ごとのエリア(獣舎・展示場)

```ts
interface Animal {
  // 既存フィールドに加えて
  enclosure?: {
    rect?: { x: number; y: number; w: number; h: number }
    points?: Array<{ x: number; y: number }>
  }
}
```

- 動物ごとに獣舎・展示場の区画をマップ上に描画できるようにする。所属エリアの内側に、
  エリア色より一段濃い塗り+縁取りで小区画として表示し、区画名(動物名)は
  ズームすると読める程度の小さなラベルで添える
- 獣舎のタップはピンのタップと同じ挙動(ポップアップ表示)
- `position` は従来どおりピン位置。enclosure がある場合、ピンは区画の中に置く
- **王子動物園は全 17 種に enclosure を定義**する(ゾウ舎、カバ舎、レッサーパンダ舎(カバ舎西隣)、
  コアラ舎、ペンギンプール、円形猛獣舎内の各獣舎、など)。架空2園は代表的な数頭でデモすればよい

### 8.3 種の情報・個体情報

```ts
interface Animal {
  // 既存フィールドに加えて
  speciesInfo?: {
    scientificName?: string   // 学名
    englishName?: string      // 英名
    distribution?: string     // 分布
    habitat?: string          // 生息環境
    diet?: string             // 食性
    lifespan?: string         // 寿命(目安)
  }
  individuals?: Array<{
    name: string              // 個体名(例: ズゼ)
    kana?: string
    sex?: 'オス' | 'メス'
    birthDate?: string        // 'YYYY-MM-DD' または 'YYYY'。年齢は表示時に計算する
    birthplace?: string       // 生まれた場所
    father?: string           // 父親の個体名
    mother?: string           // 母親の個体名
    arrivedAt?: string        // 来園年(任意)
    note?: string             // 補足(エピソードなど)
  }>
}
```

- **動物詳細ページ**に「種の情報」セクション(speciesInfo を表形式)と、
  individuals がある場合は「この園にいる個体」セクション(個体ごとのカード)を表示する
- 年齢は `birthDate` から表示時点で計算する(`YYYY` のみの場合は「約N歳」)
- 全動物に speciesInfo を付ける(全園)

### 8.4 実在園(王子動物園)の個体情報の収録方針

- **公開情報で確認できた個体・項目のみ**収録する。未確認の項目(父母など)は省略する
- 収録する個体(2026年7月時点の公開情報):
  - アジアゾウ **ズゼ**: メス、1990-04-05、ラトビア・リガ動物園生まれ、1996年来園
    (阪神・淡路大震災後、姉妹都市リガから子どもたちを元気づけるため来園)
  - アジアゾウ **マック**: オス、1992-06-13、スイス生まれ、1995年来園
  - ホッキョクグマ **ゆめ**: メス(公開情報で確認できた範囲で記載)
- disclaimer の文言に、個体情報も「収録時点の公開情報に基づき、最新の飼育状況と異なる場合がある」
  旨が伝わるようにする
- 架空2園には、父・母・生まれた場所まで含むフルスキーマの個体を数頭定義し、機能のデモとする

### 8.5 受け入れ基準(フェーズ3)

1. マップがドラッグでパン、ホイール/ボタンでズームでき、375px 幅でも操作できる
2. ドラッグ後に指/ポインタを離してもタップと誤認せずポップアップが開かない
3. `?focus=` 遷移で該当ピンへ自動ズームされる
4. 王子動物園の全動物に獣舎区画が表示され、タップでポップアップが開く
5. 動物詳細に「種の情報」が表示され、個体がいる動物には個体カード(名前・年齢・生まれた場所など)が表示される
6. 既存機能(検索・絞り込み・既存2園)に回帰がない
7. `npm run build` 成功
